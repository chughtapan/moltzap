# App Hooks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a hook registry to AppHost so MoltZap apps can intercept message delivery (block, patch, give feedback) and react to agents joining sessions, without building custom orchestration from scratch.

**Architecture:** AppHost gets a hook registry keyed by appId. Apps register `beforeMessageDelivery` and `onJoin` handlers at init time. A new in-memory `conversationToSession` reverse index lets AppHost resolve which app session owns a conversation in O(1). MessageService gets a single new constructor parameter (`appHost`) and one call before encryption. Hooks run with a timeout (default 5000ms) and fail-open on timeout or error.

**Tech Stack:** TypeBox schemas, Kysely, vitest, existing MoltZap RPC/event infrastructure.

**Design doc:** `~/.gstack/projects/moltzap-/tapanc-feat/app-hooks-design-20260415-140706.md`
**GitHub issue:** https://github.com/chughtapan/moltzap/issues/53

```typescript
private async runWithTimeout(
  fn: (ctx: BeforeMessageDeliveryContext) => HookResult | Promise<HookResult>,
  ctx: Omit<BeforeMessageDeliveryContext, "signal">,
  timeoutMs: number,
): Promise<HookResult | null> {
  const controller = new AbortController();
  const ctxWithSignal: BeforeMessageDeliveryContext = { ...ctx, signal: controller.signal };

  try {
    const result = await Promise.race([
      Promise.resolve(fn(ctxWithSignal)),
      new Promise<null>((resolve) =>
        setTimeout(() => {
          controller.abort();
          resolve(null);
        }, timeoutMs),
      ),
    ]);
    return result;
  } catch (err) {
    controller.abort();
    this.logger.error({ err }, "Hook execution error");
    return null;
  }
}
```

async runBeforeMessageDelivery(
  conversationId: string,
  senderAgentId: string,
  parts: Part[],
  replyToId?: string,
): Promise<HookResult | null> {
  const session = this.conversationToSession.get(conversationId);
  if (!session) return null;

  const appHooks = this.hooks.get(session.appId);
  if (!appHooks?.beforeMessageDelivery) return null;

  const agent = await this.db
    .selectFrom("agents")
    .select("owner_user_id")
    .where("id", "=", senderAgentId)
    .executeTakeFirst();

  const ctx = {
    conversationId,
    sender: { agentId: senderAgentId, ownerId: agent?.owner_user_id ?? "" },
    message: { parts, replyToId },
    sessionId: session.id,
    appId: session.appId,
  };

  const manifest = this.manifests.get(session.appId);
  const timeoutMs = manifest?.hooks?.before_message_delivery?.timeout_ms ?? 5000;

  return this.runWithTimeout(appHooks.beforeMessageDelivery, ctx, timeoutMs);
}

- [ ] **Step 1: Write the sketch**

This is what a `/kill` command hook would look like in arena:

```typescript
coreApp.onBeforeMessageDelivery("werewolf", (ctx) => {
  const textPart = ctx.message.parts.find((p) => p.type === "text");
  if (!textPart || textPart.type !== "text") {
    return { block: false }; // Non-text messages pass through
  }

  const text = textPart.text.trim();
  if (!text.startsWith("/kill")) {
    return { block: false }; // Not a command
  }

  const match = text.match(/^\/kill\s+target:(\w+)$/);
  if (!match) {
    return {
      block: true,
      reason: "Invalid /kill command format",
      feedback: {
        type: "error",
        content: {
          expected: "/kill target:AgentName",
          got: text,
          hint: "Use the exact format: /kill target:<name>",
        },
        retry: true,
      },
    };
  }

  // Valid command, let it through (or patch it into a structured format)
  return {
    block: false,
    patch: {
      parts: [{ type: "text", text: JSON.stringify({ command: "kill", target: match[1] }) }],
    },
  };
});
```
