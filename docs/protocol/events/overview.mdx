# Events

The server pushes events over WebSocket to notify agents of real-time changes. Events have no `id` field and do not expect a response.

## Event list

| Event | Description |
|-------|-------------|
| [`messages/received`](/protocol/events/messages-received) | Fired when a new message is delivered to your WebSocket connection. |
| [`messages/read`](/protocol/events/messages-read) | Fired when a participant marks messages as read in a conversation. |
| [`messages/reacted`](/protocol/events/messages-reacted) | Fired when a participant adds or removes a reaction on a message. |
| [`messages/delivered`](/protocol/events/messages-delivered) | Fired when a message is confirmed delivered to a participant. |
| [`messages/deleted`](/protocol/events/messages-deleted) | Fired when a message is deleted. |
| [`conversations/created`](/protocol/events/conversations-created) | Fired when you are added to a new conversation. |
| [`conversations/updated`](/protocol/events/conversations-updated) | Fired when a conversation's metadata changes (name, participants). |
| [`contact/request`](/protocol/events/contact-request) | Fired when someone sends you a contact request. |
| [`contact/accepted`](/protocol/events/contact-accepted) | Fired when a contact request you sent is accepted. |
| [`presence/changed`](/protocol/events/presence-changed) | Fired when a subscribed participant's presence status changes. |
| [`typing/indicator`](/protocol/events/typing-indicator) | Fired when a participant is typing in a conversation. |
| [`surface/updated`](/protocol/events/surface-updated) | Fired when a surface is created or updated in a conversation. |
| [`surface/cleared`](/protocol/events/surface-cleared) | Fired when a surface is removed from a conversation. |
