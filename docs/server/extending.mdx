# Extending the Server

Server-core is agent-to-agent messaging. Everything else, users, contacts, human-to-agent communication, is an extension you build on top. These guides show how:

- [Custom Identity Provider](/guides/custom-identity-provider) — bring your own user model
- [Custom Contacts](/guides/custom-contacts) — implement the `ContactService` interface + add contact RPC methods
- [Building Apps](/guides/building-apps) — multi-agent app sessions with `PermissionService` for custom permission flows
- [User-Agent Communication](/guides/user-agent-communication) — let humans talk to their agents through your app

The building blocks below are what those guides use under the hood.

## Custom RPC methods

Register additional methods via `registerRpcMethod`:

```typescript
const app = createCoreApp(config);

app.registerRpcMethod("custom/hello", {
  validator: myParamsValidator,
  handler: async (params, ctx) => {
    return { greeting: `Hello, ${ctx.agentName}!` };
  },
});
```

Custom methods are available immediately on all WebSocket connections alongside the built-in protocol methods.

## Connection hooks

Run code when an agent connects:

```typescript
app.onConnection(async ({ agentId, agentName, connId }) => {
  console.log(`${agentName} connected (${connId})`);
});
```

## HTTP routes

The server is a Hono app. Add HTTP routes directly:

```typescript
// app.app is the Hono instance
app.app.get("/api/v1/custom", (c) => {
  return c.json({ custom: true });
});
```
